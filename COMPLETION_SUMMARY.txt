RELATIONSHIPS:

  ├── bookings (customer_id, provider_id)
  ├── reviews (reviewer_id, provider_id)
  ├── conversations (participant_1_id, participant_2_id)
  └── messages (sender_id)

================================================================================
  AUTHENTICATION FLOW
================================================================================

REGISTRATION (3 STEPS):
  Step 1: Phone Entry
    → User enters phone number
    → API: POST /api/auth/send-phone-otp
    → OTP generated and stored

  Step 2: OTP Verification
    → User enters 6-digit OTP
    → API: POST /api/auth/verify-phone-otp
    → OTP validated and marked verified

  Step 3: Profile Completion
    → User enters name, email, role
    → API: POST /api/auth/complete-phone-registration
    → User profile created
    → User automatically logged in

LOGIN (2 STEPS):
  Step 1: Phone Entry
    → User enters phone number
    → API: POST /api/auth/send-phone-otp
    → OTP generated and stored

  Step 2: OTP Verification
    → User enters 6-digit OTP
    → API: POST /api/auth/phone-login
    → User logged in

   → Visit https://supabase.com
   → Create new project
   → Copy URL and Anon Key

2. ADD ENVIRONMENT VARIABLES
   → Create .env.local
   → Add NEXT_PUBLIC_SUPABASE_URL
